# WISM 화면(Screen) 플로우

## 1. 최종 화면 목록 (P-01 ~ P-09)

| ID | 화면명 | 유형 | 출처 |
|----|--------|------|------|
| P-01 | 로그인 (Splash/Login) | 풀스크린 | 기능정의서 |
| P-02 | 홈 대시보드 (Dashboard) | 탭 1 | 프로토타입 |
| P-03 | 전체 게시판 (All Memos) | 탭 2 | 기능정의서 + 프로토 |
| P-04 | 내 메모 (My Memos) | 탭 3 | 프로토타입 |
| P-05 | 북마크 (Bookmarks) | 탭 4 | 프로토타입 |
| P-06 | 프로필/설정 (Profile) | 탭 5 | 프로토 + 기능정의서 |
| P-07 | 게시글 작성/수정 (Write/Edit) | 모달 (FAB) | 기능정의서 + 프로토 |
| P-08 | 게시글 상세 (Detail + Comments) | 모달 | 기능정의서 + 프로토 |
| P-09 | 알림 패널 (Notifications) | 슬라이드 패널 | 기능정의서 + 프로토 |

---

## 2. 전체 앱 플로우

```
[앱 실행]
    │
    ▼
[스플래시] ─── 자동 로그인 + 토큰 확인
    │
    ├── 자동 로그인 ON & 토큰 유효 ──→ [P-02 홈 대시보드]
    │
    └── 토큰 없음/만료
                    ▼
             [P-01 로그인]
                    │ 인증 성공
                    ▼
             [P-02 홈 대시보드]
             (Bottom Tab 5탭 + FAB)
                    │
        ┌───────┬──┼───────┬────────┐
        ▼       ▼  ▼       ▼        ▼
     Tab 1:  Tab 2: Tab 3: Tab 4:  Tab 5:
      홈     전체   내메모  북마크   프로필
     (P-02) (P-03) (P-04) (P-05)  (P-06)

     + FAB(플로팅 버튼) → P-07 글 작성
     + 헤더 알림 버튼   → P-09 알림 패널
```

**로직:**
- 앱 실행 시 스플래시 → 자동 로그인 확인 → 로그인 화면

- 자동 로그인 체크 시 DataStore에 플래그 저장, 다음 실행부터 P-02 직행
- 로그인 실패 시 토스트 에러 메시지
- ID/PW 미입력 시 로그인 버튼 비활성화

### P-03. 전체 게시판 (All Memos) — Tab 2

```
┌─────────────────────────┐
│ 전체 메모            🔔³  │  ← 헤더
│─────────────────────────│
│ 🔍 검색어를 입력하세요    │  ← 통합 검색바
│─────────────────────────│
│ [카테고리 ▼] [우선순위 ▼] │  ← 필터 드롭다운 2개
│         [초기화]         │  ← 필터 활성 시 초기화 버튼
│─────────────────────────│
│ 12개의 메모              │  ← 결과 카운트
│ ┌─────────────────────┐ │
│ │ 🔴긴급 | 이슈 | A프로젝트│ │  ← 뱃지 (우선순위/카테고리/프로젝트)
│ │ 서버 장애 긴급 보고    │ │     제목
│ │ 오전 10시 서버 다운... │ │     미리보기 (2줄)
│ │ @담당자태그             │ │     태그
│ │ 박팀장 · 2시간전       │ │     작성자 · 일시
│ │ 👁 3/5  💬 7      🔖  │ │     읽음 · 댓글수 · 북마크
│ └─────────────────────┘ │
│ ┌─────────────────────┐ │
│ │ 일반 | 일정            │ │
│ │ 3월 월간회의 안건      │ │
│ │ 회의 안건을 공유...    │ │
│ │ 김팀장 · 5시간전       │ │
│ │ 👁 5/5  💬 3      🔖  │ │
│ └─────────────────────┘ │
│         ...              │  ← 무한 스크롤 (LazyColumn)
│                    [✏️] │  ← FAB
│─────────────────────────│
│ [홈][전체][내글][북마크][프로필]│
└─────────────────────────┘
```

**로직:**
- 최신순 정렬 (긴급 메모 상단 고정)
- 무한 스크롤 (LazyColumn + 페이지네이션)
- 당겨서 새로고침
- 검색: 제목 + 내용 실시간 검색
- 필터: 카테고리 (전체/일정/이슈/결정사항/회의록/기타) + 우선순위 (전체/긴급/일반)
- 카드 클릭 → P-08(상세)
- 북마크 아이콘 클릭 → 즐겨찾기 토글
- FAB 클릭 → P-07(글 작성)

### P-07. 게시글 작성/수정 (Write/Edit) — 모달

```
┌─────────────────────────┐
│ ✕ 취소  새 메모 작성  저장 │  ← (수정 시: "메모 수정")
│─────────────────────────│
│                         │
│ ┌───────────────────┐   │
│ │ 제목 *             │   │  ← 제목 (필수)
│ └───────────────────┘   │
│                         │
│ ┌────────┐ ┌────────┐   │
│ │우선순위▼│ │카테고리▼│   │  ← 2열: 긴급/일반, 일정/이슈/결정/회의록/기타
│ └────────┘ └────────┘   │
│                         │
│ ┌───────────────────┐   │
│ │ 프로젝트명          │   │  ← 프로젝트 (선택)
│ └───────────────────┘   │
│                         │
│ ┌───────────────────┐   │
│ │                   │   │
│ │ 내용을 입력하세요   │   │  ← 본문 (Multi-line)
│ │                   │   │
│ └───────────────────┘   │
│                         │
│ 태그                     │
│ ┌──────────────┐ [추가]  │  ← 태그 입력 + 추가 버튼
│ └──────────────┘        │
│ [×태그1] [×태그2]        │  ← 추가된 태그 (삭제 가능)
│                         │
│ ┌─────────┐┌─────────┐  │
│ │  취소    ││   저장   │  │
│ └─────────┘└─────────┘  │
└─────────────────────────┘
```

**로직:**
- 신규: FAB에서 진입, 빈 폼
- 수정: P-08 상세에서 수정 버튼 클릭 → 기존 데이터 로드
- 필수: 제목, 카테고리
- 우선순위: 긴급 / 일반 (기본값: 일반)
- 태그: 입력 후 추가 버튼으로 등록, × 클릭으로 삭제
- 저장 시 필수 항목 누락이면 토스트 경고
- 저장 완료 → 이전 화면으로 복귀 + 리스트 갱신

### P-09. 알림 패널 (Notifications) — 슬라이드 패널

```
                    ┌─────────────────────┐
                    │ 🔔 알림    ³    ✕   │  ← 안읽음 카운트 + 닫기
                    │─────────────────────│
                    │                     │
                    │ ● 💬 새 댓글         │  ← 안읽음 (배경 강조)
                    │   김팀장님이 댓글 작성  │
                    │   "확인했습니다"       │
                    │   📄 서버 장애 보고    │
                    │   5분 전              │
                    │─────────────────────│
                    │ ● 🔔 멘션             │
                    │   회의 안건에서 멘션    │
                    │   30분 전             │
                    │─────────────────────│
                    │ ○ ✅ 상태 변경         │  ← 읽음 (일반 배경)
                    │   확인 완료 처리됨     │
                    │   2시간 전            │
                    │─────────────────────│
                    │ ○ 🔴 긴급             │
                    │   긴급 이슈 등록됨     │
                    │   어제                │
                    │─────────────────────│
                    │                     │
                    │ ┌─────────────────┐ │
                    │ │ 모든 알림 읽음 표시│ │  ← 안읽음 있을 때만
                    │ └─────────────────┘ │
                    └─────────────────────┘
```

**로직:**
- 헤더 알림 아이콘 클릭 → ModalDrawerSheet 또는 커스텀 슬라이드 패널
- 알림 타입별 아이콘/색상: comment(파랑), mention(보라), status(초록), urgent(주황)
- 읽음/안읽음 시각적 구분 (배경색 + 파란 점)
- 알림 클릭 → 읽음 처리 + P-08(해당 게시글 상세) 열기
- "모든 알림 읽음 표시" 버튼 → 전체 읽음 처리
- 빈 상태: "알림이 없습니다"

## 4. 네비게이션 구조

```
[P-01 로그인]
    │ 로그인 성공
    ▼
[메인 Scaffold]
    │
    ├── Bottom Navigation (5탭)
    │   ├── Tab 1: P-02 Dashboard (홈 대시보드)
    │   │            └── → P-08 MemoDetail (상세)
```

## 8. 데이터 모델 (Kotlin)

### UI 모델 (Presentation Layer)

> Room Entity와 별도로, ViewModel → Composable 전달용 UI 모델.
> Entity ↔ UI 모델 간 매핑은 Repository에서 처리.

```kotlin
// 카테고리
enum class Category(val label: String) {
    SCHEDULE("일정"),
    ISSUE("이슈"),
    DECISION("결정사항"),
    MEETING("회의록"),
    OTHER("기타")
}

// 우선순위
enum class Priority(val label: String) {
    URGENT("긴급"),
    NORMAL("일반")
}

// 알림 타입
enum class NotificationType {
    COMMENT, MENTION, STATUS, URGENT
}
```

```kotlin
// 게시글 (목록/상세용)
data class Post(
    val id: Int,
    val title: String,
    val content: String,
    val category: Category,
    val priority: Priority,
    val author: Author,
    val project: String? = null,
    val tags: List<String> = emptyList(),
    val commentCount: Int = 0,
    val readBy: Int = 0,
    val totalReaders: Int = 0,
    val isRead: Boolean = false,
    val isBookmarked: Boolean = false,
    val isMine: Boolean = false,
    val comments: List<Comment> = emptyList(),
    val createdAt: String,
    val updatedAt: String
)

// 작성자 정보
data class Author(
    val id: Int,
    val name: String,
    val department: String? = null,
    val position: String? = null
)

// 댓글
data class Comment(
    val id: Int,
    val author: Author,
    val content: String,
    val type: String = "comment",   // "comment" | "status"
    val createdAt: String
)

// 알림
data class Notification(
    val id: Int,
    val type: NotificationType,
    val title: String,
    val content: String,
    val memoTitle: String? = null,
    val postId: Int? = null,
    val sender: Author? = null,
    val isRead: Boolean = false,
    val createdAt: String
)

// 사용자 프로필
data class UserProfile(
    val id: Int,
    val loginId: String,
    val name: String,
    val department: String? = null,
    val position: String? = null,
    val email: String? = null,
    val phone: String? = null,
    val photoUrl: String? = null,
    val role: String = "manager",
    val stats: UserStats = UserStats(),
    val settings: UserSettings = UserSettings()
)

data class UserStats(
    val postCount: Int = 0,
    val commentCount: Int = 0,
    val bookmarkCount: Int = 0
)

data class UserSettings(
    val pushEnabled: Boolean = true,
    val autoLoginEnabled: Boolean = false
)
```
