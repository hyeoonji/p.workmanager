# WISM DB 스키마 설계

## 0. 개요

- **Phase 1 (현재)**: Room DB (SQLite) - 동일한 스키마를 Room Entity로 구현. Seed 테스트 데이터 포함.
- **Phase 2 (서버 연동)**: MySQL - 아래 SQL 스키마 그대로 사내 서버에 적용.
- **네이밍 규칙**: 스네이크 케이스 (snake_case), WISM 전용 테이블은 `wism_` 접두어
- **공통 컬럼**: `created_at`, `updated_at`은 모든 테이블에 포함
- **외래키**: user 참조는 `user_id`로 통일

> Room Entity 클래스는 동일한 테이블명을 사용하여 Phase 2 MySQL 스키마와 1:1 대응.

## 2. 테이블 상세

### 2-0. wism_users (사용자)

> Phase 1: Room Entity로 로컬 저장 (Seed 데이터).
> Phase 2: 사내 서버 연동 시 이 테이블 대신 기존 사내 users 테이블을 참조하도록 전환.

```sql
CREATE TABLE wism_users (
    id          INT AUTO_INCREMENT PRIMARY KEY,
    login_id    VARCHAR(50)  NOT NULL UNIQUE,         -- 로그인 아이디
    password    VARCHAR(255) NOT NULL,                 -- bcrypt 해시 (Phase 2) / 평문 (Phase 1)
    name        VARCHAR(50)  NOT NULL,                 -- 이름
    department  VARCHAR(100) DEFAULT NULL,              -- 부서 (예: XR개발실 3팀)
    position    VARCHAR(50)  DEFAULT NULL,              -- 직급 (예: 팀장, 수석, 임원)
    email       VARCHAR(100) DEFAULT NULL,              -- 이메일
    phone       VARCHAR(20)  DEFAULT NULL,              -- 연락처
    photo_url   VARCHAR(500) DEFAULT NULL,              -- 프로필 사진 URL
    role        ENUM('admin', 'manager') DEFAULT 'manager',  -- 권한 등급
    is_active   TINYINT(1)   DEFAULT 1,                -- 활성 상태 (1=활성, 0=비활성)
    created_at  DATETIME     DEFAULT CURRENT_TIMESTAMP,
    updated_at  DATETIME     DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

| 컬럼 | 타입 | 필수 | 설명 |
|------|------|------|------|
| id | INT (PK) | O | 사용자 고유 ID |
| login_id | VARCHAR(50) | O | 로그인 아이디 (UNIQUE) |
| password | VARCHAR(255) | O | 비밀번호 (Phase 1: 평문, Phase 2: bcrypt) |
| name | VARCHAR(50) | O | 이름 |
| department | VARCHAR(100) | - | 부서명 |
| position | VARCHAR(50) | - | 직급 |
| email | VARCHAR(100) | - | 이메일 |
| phone | VARCHAR(20) | - | 연락처 |
| photo_url | VARCHAR(500) | - | 프로필 사진 URL |
| role | ENUM | - | 권한 (admin/manager) |
| is_active | TINYINT | - | 계정 활성 상태 |

### 2-1. wism_posts (게시글/메모)

```sql
CREATE TABLE wism_posts (
    id          INT AUTO_INCREMENT PRIMARY KEY,
    user_id     INT          NOT NULL,                 -- 작성자
    category    ENUM('schedule','issue','decision','meeting','other') NOT NULL,
    priority    ENUM('urgent','normal') DEFAULT 'normal',  -- 우선순위
    title       VARCHAR(200) NOT NULL,                 -- 제목
    content     TEXT         NOT NULL,                 -- 본문
    project     VARCHAR(100) DEFAULT NULL,              -- 프로젝트명
    is_deleted  TINYINT(1)   DEFAULT 0,                -- 소프트 삭제
    created_at  DATETIME     DEFAULT CURRENT_TIMESTAMP,
    updated_at  DATETIME     DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,

    INDEX idx_user_id (user_id),
    INDEX idx_category (category),
    INDEX idx_priority (priority),
    INDEX idx_created_at (created_at DESC),
    INDEX idx_deleted_created (is_deleted, created_at DESC),
    FOREIGN KEY (user_id) REFERENCES wism_users(id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

| 컬럼 | 타입 | 필수 | 설명 |
|------|------|------|------|
| id | INT (PK) | O | 게시글 고유 ID |
| user_id | INT (FK) | O | 작성자 ID |
| category | ENUM | O | schedule(일정) / issue(이슈) / decision(결정사항) / meeting(회의록) / other(기타) |
| priority | ENUM | - | urgent(긴급) / normal(일반). 기본값: normal |
| title | VARCHAR(200) | O | 제목 |
| content | TEXT | O | 본문 내용 |
| project | VARCHAR(100) | - | 관련 프로젝트명 |
| is_deleted | TINYINT | - | 소프트 삭제 플래그 |

## 4. Phase 전환 가이드

### Phase 1 → Phase 2 전환 시 변경사항

| 항목 | Phase 1 (현재) | Phase 2 (서버 연동) |
|------|---------------|-------------------|
| **데이터 소스** | Room (LocalDataSource) | Retrofit (RemoteDataSource) |
| **Hilt 모듈** | DatabaseModule → Room DAO 제공 | NetworkModule → Retrofit ApiService 제공 |
| **Repository** | 인터페이스 동일, 구현체만 교체 | 인터페이스 동일, 구현체만 교체 |
| **wism_users** | Room Entity (Seed 데이터) | 사내 DB users 테이블 참조 |
| **인증** | DataStore (로컬 로그인) | JWT 토큰 (서버 발급) |
| **비밀번호** | 평문 비교 | bcrypt 해시 (서버 검증) |
| **wism_fcm_tokens** | 미사용 | 활성화 (FCM 푸시) |

### 사내 서버 DB 전환 시 추가 변경

| 항목 | 변경 내용 |
|------|----------|
| **wism_users** | 사용 중지. 기존 사내 users 테이블로 교체 |
| **API User 쿼리** | `FROM wism_users` → `FROM [사내 테이블명]` |
| **컬럼 매핑** | 사내 테이블 컬럼명에 맞춰 매핑 수정 |
| **FK 참조** | user_id가 사내 테이블 PK 참조하도록 변경 |
| **나머지 테이블** | 변경 없음. 그대로 사내 DB에 생성 |

## 5. 초기화 SQL (Phase 2 서버용)

```sql
-- ============================================
-- WISM Database Schema v2
-- Phase 2 서버 배포 시 사용
-- ============================================

CREATE TABLE IF NOT EXISTS wism_users (
    id          INT AUTO_INCREMENT PRIMARY KEY,
    login_id    VARCHAR(50)  NOT NULL UNIQUE,
    password    VARCHAR(255) NOT NULL,
    name        VARCHAR(50)  NOT NULL,
    department  VARCHAR(100) DEFAULT NULL,
    position    VARCHAR(50)  DEFAULT NULL,
    email       VARCHAR(100) DEFAULT NULL,
    phone       VARCHAR(20)  DEFAULT NULL,
    photo_url   VARCHAR(500) DEFAULT NULL,
    role        ENUM('admin','manager') DEFAULT 'manager',
    is_active   TINYINT(1)   DEFAULT 1,
    created_at  DATETIME     DEFAULT CURRENT_TIMESTAMP,
    updated_at  DATETIME     DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

CREATE TABLE IF NOT EXISTS wism_posts (
    id          INT AUTO_INCREMENT PRIMARY KEY,
    user_id     INT          NOT NULL,
    category    ENUM('schedule','issue','decision','meeting','other') NOT NULL,
    priority    ENUM('urgent','normal') DEFAULT 'normal',
    title       VARCHAR(200) NOT NULL,
    content     TEXT         NOT NULL,
    project     VARCHAR(100) DEFAULT NULL,
    is_deleted  TINYINT(1)   DEFAULT 0,
    created_at  DATETIME     DEFAULT CURRENT_TIMESTAMP,
    updated_at  DATETIME     DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    INDEX idx_user_id (user_id),
    INDEX idx_category (category),
    INDEX idx_priority (priority),
    INDEX idx_created_at (created_at DESC),
    INDEX idx_deleted_created (is_deleted, created_at DESC),
    FOREIGN KEY (user_id) REFERENCES wism_users(id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

CREATE TABLE IF NOT EXISTS wism_comments (
    id          INT AUTO_INCREMENT PRIMARY KEY,
    post_id     INT          NOT NULL,
    user_id     INT          NOT NULL,
    content     TEXT         NOT NULL,
    type        ENUM('comment','status') DEFAULT 'comment',
    is_deleted  TINYINT(1)   DEFAULT 0,
    created_at  DATETIME     DEFAULT CURRENT_TIMESTAMP,
    updated_at  DATETIME     DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    INDEX idx_post_id (post_id),
    INDEX idx_user_id (user_id),
    FOREIGN KEY (post_id) REFERENCES wism_posts(id),
    FOREIGN KEY (user_id) REFERENCES wism_users(id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

CREATE TABLE IF NOT EXISTS wism_tags (
    id          INT AUTO_INCREMENT PRIMARY KEY,
    name        VARCHAR(100) NOT NULL UNIQUE,
    created_at  DATETIME     DEFAULT CURRENT_TIMESTAMP
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

CREATE TABLE IF NOT EXISTS wism_post_tags (
    post_id     INT NOT NULL,
    tag_id      INT NOT NULL,
    PRIMARY KEY (post_id, tag_id),
    FOREIGN KEY (post_id) REFERENCES wism_posts(id),
    FOREIGN KEY (tag_id)  REFERENCES wism_tags(id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

CREATE TABLE IF NOT EXISTS wism_bookmarks (
    user_id     INT NOT NULL,
    post_id     INT NOT NULL,
    created_at  DATETIME DEFAULT CURRENT_TIMESTAMP,
    PRIMARY KEY (user_id, post_id),
    INDEX idx_user_id (user_id),
    FOREIGN KEY (user_id) REFERENCES wism_users(id),
    FOREIGN KEY (post_id) REFERENCES wism_posts(id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

CREATE TABLE IF NOT EXISTS wism_read_receipts (
    user_id     INT NOT NULL,
    post_id     INT NOT NULL,
    read_at     DATETIME DEFAULT CURRENT_TIMESTAMP,
    PRIMARY KEY (user_id, post_id),
    INDEX idx_post_id (post_id),
    FOREIGN KEY (user_id) REFERENCES wism_users(id),
    FOREIGN KEY (post_id) REFERENCES wism_posts(id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;

CREATE TABLE IF NOT EXISTS wism_notifications (
    id          INT AUTO_INCREMENT PRIMARY KEY,
    user_id     INT          NOT NULL,
    type        ENUM('comment','mention','status','urgent') NOT NULL,
    post_id     INT          DEFAULT NULL,
    comment_id  INT          DEFAULT NULL,
    sender_id   INT          DEFAULT NULL,
    title       VARCHAR(100) NOT NULL,
    content     VARCHAR(255) NOT NULL,
    memo_title  VARCHAR(200) DEFAULT NULL,
    is_read     TINYINT(1)   DEFAULT 0,
```

## 6. Room Entity 매핑 (Phase 1)

### MySQL → Room 변환 규칙

| MySQL | Room (Kotlin) | 비고 |
|-------|--------------|------|
| `INT AUTO_INCREMENT PRIMARY KEY` | `@PrimaryKey(autoGenerate = true) val id: Int = 0` | |
| `ENUM('a','b','c')` | `val field: String` | Room은 ENUM 미지원, 앱 레벨에서 검증 |
| `DATETIME DEFAULT CURRENT_TIMESTAMP` | `val createdAt: String` | ISO 8601 문자열 (예: "2025-03-22T10:30:00") |
| `TINYINT(1) DEFAULT 0` | `val field: Boolean = false` | Room TypeConverter 사용 |
| `TEXT` | `val field: String` | |
| `VARCHAR(n)` | `val field: String` | Room에서 길이 제한 없음, 앱 레벨 검증 |
| `FOREIGN KEY` | `@ForeignKey(...)` | Entity 어노테이션에 정의 |
| `INDEX` | `@Entity(indices = [...])` | |
| `DEFAULT NULL` | `val field: String? = null` | nullable |
